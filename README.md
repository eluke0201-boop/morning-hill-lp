# Morning Hill — 朝活カフェ ランディングページ

> 架空の朝活カフェ「Morning Hill」のLP制作。**ブリーフ作成から実装・画像生成まで一気通貫**で行ったポートフォリオ作品です。

- **Live**: https://morning-hill-lp.vercel.app
- **Source**: https://github.com/eluke0201-boop/morning-hill-lp

## ✨ 制作のポイント

- **ブリーフ起点の設計** — 架空クライアントの設定（業態・ターゲット・差別化）を [BRIEF.md](BRIEF.md) に先に固め、すべてのデザイン判断をそこから導く
- **AIで20枚の画像を生成** — OpenAI の画像生成モデルで、写真とアイコンを統一トーンで一括生成。ブランド方針の変更に強い設計に
- **レスポンシブ対応** — スマホからPCまで、3段階のブレークポイントで崩れずに表示
- **軽量実装** — フレームワーク不使用。Vanilla HTML/CSS/JS のみで、依存ゼロ・読み込み速度優先

## 🎨 デザインコンセプト

ブランドのキーワードは **「朝の光・ミニマル・温かみ」**。

クリームベースに朝陽オレンジを差し色とした配色、欧文セリフ＋和文サンセリフのフォントの組み合わせで「丁寧に淹れたコーヒー」のような世界観をWebに落とし込みました。文字より「空気」を見せる、余白を大きく取った構成。

## 🧱 構成

1ページ完結のランディングページ（7セクション）：Hero / About / Menu / Space / Community / Access / Footer。

## 🤖 AI画像生成の工夫

すべての画像を Python スクリプトで一括生成。ポイントは「写真用」「フラットアイコン用」の2系統のスタイル指定を切り替えられるようにし、20枚全体のトーンを揃えたこと。1コマンドで全画像を再生成できるため、ブランドの方向性が変わっても作り直しは数分で完了します。

## 📝 制作プロセス

1. クライアント設定書（BRIEF.md）作成
2. 構成・情報設計
3. HTML/CSS/JS 実装
4. AI画像生成スクリプト構築 → アイコン・写真生成
5. レスポンシブ確認・調整
6. OGP画像・メタタグ・構造化データを整備
